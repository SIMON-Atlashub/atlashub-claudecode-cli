Introduction

Le workflow Business Analyse est specialise pour la specification fonctionnelle et technique. Il combine la methodologie BABOK v3 avec une approche systematique pour produire des specifications completes et autonomes. Le BA ne code jamais - il produit uniquement des documents de specification.

The Business Analyse workflow is specialized for functional and technical specification. It combines BABOK v3 methodology with a systematic approach to produce complete and autonomous specifications. The BA never codes - it only produces specification documents.

⚠
Principe fondamental
Core Principle

LE BA NE CODE JAMAIS. Il produit des specifications suffisamment detaillees pour qu'un developpeur puisse implementer de maniere autonome, sans poser de questions.

THE BA NEVER CODES. It produces specifications detailed enough for a developer to implement autonomously, without asking questions.

💡
Questions adaptatives
Adaptive Questions

Le nombre de questions s'adapte a la complexite : 6-8 pour un CRUD simple, jusqu'a 30+ pour une feature critique. Plus besoin de repondre a 44 questions pour un simple formulaire !

The number of questions adapts to complexity: 6-8 for a simple CRUD, up to 30+ for a critical feature. No more answering 44 questions for a simple form!
