BA - Business Analysis

Workflow complet pour applications Microsoft (.NET/Blazor/EF Core) Complete workflow for Microsoft applications (.NET/Blazor/EF Core)

Introduction

Le workflow BA (Business Analysis) est specialise pour les applications Microsoft stack (.NET, Blazor, EF Core). Il combine l'approche EPCT (Explore-Plan-Code-Test) avec une analyse fonctionnelle rigoureuse et un challenge agressif du besoin metier.

The BA (Business Analysis) workflow is specialized for Microsoft stack applications (.NET, Blazor, EF Core). It combines the EPCT approach (Explore-Plan-Code-Test) with rigorous functional analysis and aggressive business requirement challenging.

📊
Stack Microsoft supporte
Supported Microsoft Stack

.NET 6/7/8, ASP.NET Core Web API, Blazor Server/WASM, Entity Framework Core 6+, SQL Server, Identity

.NET 6/7/8, ASP.NET Core Web API, Blazor Server/WASM, Entity Framework Core 6+, SQL Server, Identity

Agents BA BA Agents

Le workflow BA utilise des agents specialises pour optimiser les couts et la qualite :

The BA workflow uses specialized agents to optimize costs and quality:

Agent Model Usage Usage Cout Cost
explore-codebase Haiku Scan projet, lecture entites Project scan, entity reading Low
ba/entity-analyzer Haiku Analyse relations EF Core EF Core relations analysis Low
ba/doc-generator Sonnet Generation documents Document generation Medium
snipper Opus Generation code qualite Quality code generation High
💡
Strategie de couts
Cost strategy

Les phases de lecture utilisent Haiku (low cost), tandis que la generation de code utilise Opus (high cost) pour garantir la qualite.

Reading phases use Haiku (low cost), while code generation uses Opus (high cost) to ensure quality.
